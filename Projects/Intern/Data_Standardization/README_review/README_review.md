### 1. 가치를 먼저 제공하기

**원본**
```

표준 단어·용어·코드 체계 구축과 협업 프로세스를 개선하는 프로젝트를 수행했습니다.
다양한 인스턴스와 서비스에서 동일한 의미의 데이터가 서로 다른 방식으로 정의되어
중복, 불일치, 의미 왜곡이 발생하는 문제를 해결하고자 했습니다.
```

**문제**  
- 두 문장 모두 "무엇을 했는지"에만 집중하고 있습니다.
- 읽는 사람이 이 문서에서 무엇을 얻을 수 있는지 드러나지 않습니다.
- 문제 상황 설명이 마지막 문장에 있어서, 이 프로젝트가 왜 필요했는지를 바로 파악하기 어렵습니다.

**개선**
```
서로 다른 시스템에서 같은 데이터가 다르게 정의되면 분석 결과가 달라지고
의사결정의 신뢰도가 낮아집니다.

이 문서는 문제를 해결하기 위한 전사 데이터 표준화 프로젝트의
배경, 작업 방식, 결과를 설명합니다.
이 문서를 읽으면 표준화 작업이 어떻게 진행됐는지, 협업 구조를 어떻게
개선했는지 파악할 수 있습니다.
```

**개선 방향**

- 문제 상황을 먼저 제시해서 독자가 맥락을 잡을 수 있게 했습니다.
- 이 문서로 어떤 내용을 알 수 있는지를 명시해서 독자의 기대를 설정했습니다.

## 3. 표준 단어·용어·코드 사전 구축하기

**배경**  
시스템마다 동일한 컬럼이 다르게 정의되어 있어 데이터 간 연계가 어렵고,
분석 결과의 신뢰도가 낮았습니다.

**방법**

전사 DB 컬럼명을 언더스코어 기준으로 분리했습니다.
16만 건 이상의 메타데이터를 정제해 6,179개 표준화 대상을 선별했습니다.


표준 단어는 사용 빈도에 따라 두 가지 방식으로 정의했습니다.

- 여러 테이블에서 사용되는 컬럼: 포괄적 용어로 정의 (예: `단위_유형_코드`, `전체_여부`)
- 특정 테이블에서만 사용되는 컬럼: 테이블 맥락을 반영한 구체적 용어로 정의


`_코드`로 끝나는 컬럼은 실제 데이터 값 수를 확인해 코드 여부를 판별했습니다.

- 코드가 아닌 경우 `_값` 또는 `_내용`으로 재정의해 의미 정확성을 확보했습니다.

**결과**  
전사 표준 단어·용어·코드 사전을 구축했습니다.
컬럼 정의 오류를 정정해 시스템 간 데이터 일관성을 확보했습니다.
표준화된 데이터를 바탕으로 자연어 쿼리 기반 검색 품질도 향상됐습니다.

## 4. 표준 단어 후보 자동 추출하기

**배경**  
작업자가 컬럼 코멘트를 직접 보면서 단어를 정의하고 판단 기준인 단어 비율을 계산하면 시간이 많이 걸립니다.

**방법**  
Python과 [Kiwi 형태소 분석기](https://github.com/bab2min/kiwi)를 활용해 자동화 파이프라인을 개발했습니다. Kiwi는 빠른 속도와 범용적인 성능을 지향하는 한국어 형태소 분석기 라이브러리입니다. 

영어 단어를 입력하면 해당 단어가 포함된 전사 컬럼 코멘트를 분석합니다. 분석 결과를 바탕으로 가장 많이 쓰인 한글 단어를 빈도 기반으로 추천합니다.

```python
# 예시: 'type'이 포함된 컬럼 코멘트에서 한글 단어 빈도 계산

# 1. kiwi로 형태소 분리하기
from kiwipiepy import Kiwi, Match
kiwi = Kiwi()

##'type'이 포함된 컬럼의 코멘트 목록
inputs = ['생성일시', '수정일시', '종료일자', '가입일자', '계약일자']

tokens = []

for word in inputs:
    tokenization = kiwi.tokenize(word, normalize_coda=True)
    for i in range(len(tokenization)):
        tokens.append(tokenization[i][0])
print(tokens)

## 출력 결과 예시: ['생성', '일시', '수정', '일시', '종료일', '자', '가입', '일자', '계약', '일자']

# 2. 단어별 분리된 형태소 비율 계산

import pandas as pd

## 예시 데이터프레임
### dict_df: 영문 단어와 한글 단어, 그에 대한 comment_cnt
dict_df = pd.DataFrame({
    'en_word': ['receipt', 'receipt', 'receipt', 'receipt'],
    'ko_word': ['영수증', '받기', '수령액', '영수증'],
    'comment_cnt': [2, 0, 0, 1]
})

### comment_df: 다른 데이터프레임 (예시로 동일한 데이터 사용)
comment_df = pd.DataFrame({
    'en_word': ['receipt', 'receipt', 'receipt', 'receipt'],
    'ko_word': ['영수증', '받기', '수령액', '영수증'],
    'comment_cnt': [3, 0, 0, 1]
})

## 1) 두 데이터프레임을 합침
merged_df = pd.concat([dict_df, comment_df])

## 2) 'en_word'와 'ko_word'로 groupby하고 'comment_cnt'의 합을 구함
grouped_df = merged_df.groupby(['en_word', 'ko_word'], as_index=False)['comment_cnt'].sum()

## 3) 결과 출력
print(grouped_df)

```

**결과**  
표준 단어 선정의 속도와 일관성을 동시에 높였습니다.

## 5. 협업 구조 개선하기

이슈 히스토리의 부재로 중복 질의가 계속 발생했습니다.

**방법**

**결과**

- 회의 시간 약 30% 절감
- 정보 비대칭 감소로 병렬 작업 효율 향상

**일관성은 작은 것에서 무너집니다.** 'Wiki'와 '위키'처럼 혼재된 표기는 사소해 보이지만,
