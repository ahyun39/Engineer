# 메타데이터 활용성 증대를 위한 AI 어시스턴트 구축

## 1. 프로젝트 개요

### 1.1. 프로젝트명
    •	생성형 AI 기반 메타데이터 어시스턴트 구축

### 1.2. 배경 및 목표
    •	배경: 데이터 표준화 프로젝트를 통해 전사 표준 단어, 용어, 코드 등 방대한 메타데이터를 구축했습니다. 표준화 결과는 Wiki나 정적 문서 형태로 저장되어 있으며, 개발 직군 뿐만 아니라 모든 사내 구성원이 원하는 정보를 실시간으로 탐색하고 활용할 수 있도록 활용성을 높이고자 했습니다.
     
    •	목표: 생성형 AI를 활용하여 메타데이터를 지능형 지식 자산으로 활용하는 것을 목표로 했습니다. 사용자가 자연어로 질의하면 이에 관련된 데이터 정의를 찾아주는 AI 어시스턴트를 구축하여, 전사적인 데이터 접근성을 높이고 데이터 기반 의사결정을 확대하고자 했습니다.

### 1.3. 활용 AI 기술
    •	Google Gemini Gem

<br>

## 2. AI 활용 과정

### Phase 1: 지식베이스 구축
    •	데이터 수집: 표준화 프로젝트의 산출물인 표준 단어 사전, 표준 용어, 표준 코드값 파일

    •	구조화: AI가 명확하게 이해하고 참조할 수 있도록 데이터를 구조화했습니다. 예를 들어, "한글명: 상태, 영문약어: STATUS, 설명: 데이터의 현재 상태를 나타내는 표준 단어"와 같이 Key-Value 형태의 텍스트 파일로 변환하여 지식의 일관성과 정확도를 확보했습니다. 모호한 표현이나 중복된 내용은 최대한 제거하여 AI의 혼란 가능성을 최소화했습니다.

### Phase 2: 맞춤형 AI Gem 생성
    •	페르소나: AI 어시스턴트의 역할과 정체성을 "전사 데이터 표준 및 메타데이터에 대해 답변하는 데이터 관리자"로 정의했습니다.

    •	프롬프트: Gem의 행동 원칙을 구체적으로 설계했습니다. "당신은 제공된 지식베이스 파일만을 기반으로 답변해야 한다", "만약 질문에 대한 정보가 지식베이스에 없다면, '학습되지 않은 내용입니다'라고 솔직하게 답변해야 한다" 와 같은 명확한 가이드라인을 제시하여, AI가 임의의 정보를 생성하는 할루시네이션을 방지하고 답변의 신뢰도를 높이고자 했습니다.

    •	지식베이스 연동: 정제 및 구조화를 마친 지식베이스 파일을 Gem에 업로드하여 AI에 지식 데이터로 전달했습니다.

### Phase 3: 테스트 및 고도화
    •	시나리오 테스트:
        “주문 상태 코드는?”, “‘생성일시’를 나타내는 표준 영문명은?”, “컬럼명에 ‘yn’이 포함된 경우 무슨 의미인가요?” 등 실제 사용자 질의 시나리오를 기반으로 테스트를 반복했습니다.

    •	반복적 개선:
        AI가 의도를 잘못 파악하거나 불완전한 답변을 할 경우, 원인을 분석하여 프롬프트 엔지니어링과 데이터 표현 방식을 수정했습니다. 반복 학습을 통해 Gem의 답변 정확도와 일관성을 지속적으로 향상시켰습니다.

## 3. 느낀 점

### 3.1. ‘잘 정제된 데이터’의 중요성
    •	생성형 AI의 성능은 결국 입력 데이터의 품질에 의해 결정된다는 것을 느낄 수 있었습니다. 표준화된 데이터 정제 과정이 모델의 응답 품질을 좌우했고, AI가 사용자의 의도를 정확히 이해하고 신뢰도 높은 답변을 생성하는 기반이 되었습니다. AI의 성능은 사람이 설계한 데이터 구조 위에서 완성된다는 사실을 실감할 수 있었습니다.

### 3.2. 데이터를 ‘대화형 지식 자산’으로 전환
    •	단순한 정보 저장소였던 메타데이터를 생성형 AI를 통해 실시간으로 탐색 가능한 지식 시스템으로 바꿀 수 있다는 것을 경험할 수 있었습니다. 복잡한 표준 정보를 자연어로 질의하고 즉시 결과를 얻을 수 있는 구조를 구현하면서, AI가 데이터와 사용자를 연결하는 지능형 인터페이스로 작동할 수 있다는 것을 알 수 있었습니다. 기술적 혁신뿐 아니라, 사용자의 접근성을 높이고 조직의 데이터 활용성 증대에 AI가 유용할 수 있다는 것을 느낄 수 있었습니다.

### 3.3. ‘정보 제공자’를 넘어 ‘업무 파트너’로
    •	현재는 메타데이터 조회에 초점이 맞춰져 있지만, 향후 이 시스템을 활용해 자연어 질의를 SQL 쿼리로 변환하여 워크플로우에 적용하는 등 여러 방면으로 활용 가능한 기능이 구현될 수 있다는 기대를 해볼 수 있었습니다. 예를 들어, “지난주 가입한 특정 등급의 사용자 수를 알려줘”와 같은 질의에 대해 자동으로 결과를 산출하거나, Jira/Slack 등 협업 도구와 연동하여 프로젝트 논의 중 데이터 기반 인사이트를 즉시 제공하는 등 ‘AI 업무 파트너’로 사용될 수 있겠다는 생각을 할 수 있었습니다.

_평소 GPT, Gemini, Claude 등 여러 AI를 다양하게 활용하고 있지만 업무에 적용해본 경험은 처음이었습니다. 이번 경험을 통해 가장 크게 느낀 점은 AI를 활용하여 데이터의 사용 범위를 기존에 비해 더 큰 범위로 넓힐 수 있다는 것이었습니다. 표준화된 데이터를 단순히 저장하여 사용자에게 제공하는 것뿐만 아니라 AI로 사용자가 데이터를 더 쉽게 사용할 수 있게 해줌으로써 활용성을 극대화하는 것을 실현할 수 있다는 것을 경험할 수 있었습니다. AI 기술이 점차 발전되면서 업무 프로세스에 여러 방면으로 적용하며 더 다양한 인사이트를 얻어 업무 생산성도 높일 수 있겠다는 생각을 할 수 있었습니다._
